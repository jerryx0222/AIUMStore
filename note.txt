1.環境:docker,django REST(後端)+React(前端),MySQL
2.專案名稱:AIUMStore(目錄 D:\AIUMStore 不需重建)
3.預留React Native for ios,Android App
4.未來要上傳Cloud Run使用
5.網站功能:會員登出入,購物車,結帳台,電子支附,產品種類,產品細項及價格,




會員分類:
1.superuser:無使用限制
2.firm:只能修改自己店面的產品資料(如上架與否,優惠價格),觀看自己店面的營收/經營狀態等
3.member(LV1~10):依不同等級有不同的優惠及點數,各個會員可自由編輯其"喜好產品",並有自己的購買記錄,最常購買的產品
4.guest:可看所有上架的產品,產品只能現場結帳及取貨


類別:
1.class 人員:level,名字,手機,電話,EMAIL,LINE ID,地址,備註:名字可重復,用流水號當KEY(人員ID)
    1.1.class 人員.level=superuser,class 品牌主,class 店主,class 店員,class 會員
2.class 品牌:英文名,中文名,ICON,聯絡人ID(class 人員.人員ID),網址,備註
7.class 產品:產品類別,產品圖(WebP格式,內存多個圖片),建議價格,實售價格(內定同建議價格),規格,製程
3.class 產品品牌: 繼承 class品牌,數個class 產品
4.class 加盟品牌: 繼承 class品牌,數個class 產品品牌
5.class 品牌主:繼承 class 人員,單一class 加盟品牌
6.class 店主:繼承 class 人員,數個class 加盟品牌
7.class 店員:繼承 class 人員
8.class 會員:繼承 class 人員


當登入者為XXX,顯示維護頁:
1.店主:顯示其下所有的店員資料
2.加盟主:顯示所下所代理的品牌(含該品牌的產品)及店主資料
3.品牌主:顯示該品牌的所有產品
4.superuser:顯示以上所有的資料


新增PY程式檔:
1.將目前MySQL內的所有相關的資料庫匯出保存成"日期時間.sql"檔
2.將所滙出的"日期時間.sql"檔匯入本機的資料庫

backend/db_backup.py：資料庫匯出/匯入工具
python db_backup.py export → 匯出成 backups/20260706_090358.sql 這種「日期時間.sql」格式（用 mysqldump，含 schema + 資料）
python db_backup.py import [檔案] → 匯入指定檔案；不給檔名時自動抓 backups/ 底下最新的一份
backend/Dockerfile 加裝 default-mysql-client，讓容器內有 mysqldump/mysql 指令可用（已重建 image 生效）
.gitignore 加入 backend/backups/，避免備份檔（含密碼雜湊等敏感資料）被提交進版本控制
docker compose exec backend python db_backup.py export
docker compose exec backend python db_backup.py import

docker-compose exec backend python manage.py runserver
